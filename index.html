Web Accessibility

WCAG 2.0

Websites must comply with the Web Content Accesibility Guidelines

The four principles of WCAG is:

  • Perceivable
  • Operable
  • Understandable
  • Robust

Types of Impairments

Visual impairments include: blindness, partial or full, colour blindness, cataracts, glaucoma, age deterioration, etc.

Mobilitiy & dexterity impairments include: missing limbs, limited movement, difficulty with fine control, etc.

Auditory impairments include: deafness, partial deafness, etc.

Cognative impairments include: dyslexia, memory issues, hyperactivity, attention deficits, etc.

Styling text

Font properties

font-size
For adjusting the size of the font: a single number with a unit (rem [the best choice], em, px)
Ex: 2rem, 1.125em, 24px
font-weight
Sets how thick or thin characters in text should be displayed
Ex: bold, normal, 700
font-style
Specifies font style for text
Ex: italic or normal
line-height
For adjusting the line-height: a single number, a multiplier that multiplies the font-size
Ex: 1.5

Text decorations

text-transform
Change capitalization such as uppercase, lowercase, capitalize. Itâ€™s a good idea to use CSS to make all caps instead of writing it directly in HTML for accessibility reasons.
Ex: uppercase, lowercase, capitalize
text-decoration
Specifies the decoration added to text. The only things in a website that should be underlined are links.
Ex: none, underline, line-through, overline
text-align
Specifies the horizontal alignment of text in an element
Ex: left, center (American spelling), right, justify
text-indent
Indent the first line of text like traditionally printed materials
Ex: 1rem, 2em, 20px, 25%
text-shadow
Adding a shadow to the text. Specified like this: text-shadow: horizontal-offset vertical-offset blur-radius colour
text-shadow: 2px 2px #ff0000;
